## Contributing to Documentation

### Adding New Content

**New feature documented?**
1. Create section in appropriate guide
2. Add entry to this INDEX.md
3. Add screenshot placeholders if applicable
4. Update README.md if major feature

**Updating existing content?**
1. Edit the relevant guide
2. Update "Last Updated" date at bottom
3. Add note in changelog (if major change)
4. Keep screenshot placeholders in sync

### Documentation Standards

**Writing style**:
- Use clear, concise language
- Write in active voice
- Use examples liberally
- Include code blocks for commands
- Provide screenshots placeholders

**Formatting**:
- Use markdown headers (h2, h3, h4)
- Add table of contents for long documents
- Use tables for reference data
- Bold for emphasis, not italics
- Code blocks for all commands and code

**Structure**:
- Start with overview/introduction
- Provide step-by-step instructions
- Include troubleshooting section
- End with "Next Steps" links
